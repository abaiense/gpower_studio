## Visão Geral

SimplyBook.me é um sistema SaaS de agendamento online voltado para negócios baseados em serviços. A plataforma permite que empresas criem uma página de reservas totalmente personalizada, aceitem pagamentos, gerenciem equipes e se comuniquem com clientes de forma automatizada — tudo operando 24/7.

O diferencial central do produto é o sistema de **Custom Features**: em vez de cobrar por tudo em um plano fixo, a plataforma oferece um conjunto base de funcionalidades e permite que cada negócio selecione add-ons pagos conforme a necessidade. Os planos diferem pelo número de custom features incluídas e pelo volume de agendamentos mensais permitidos.

**Dados gerais:**
- 50.000+ negócios ativos
- 200+ países atendidos
- 30+ milhões de reservas processadas
- 20+ idiomas suportados
- 80+ funcionalidades disponíveis (base + custom features)
- Adquirida em setembro de 2023 pela team.blue

## Modelo de Negócio

SimplyBook.me opera como **SaaS com modelo freemium + add-ons pagos**:

1. **Plano gratuito** com 50 bookings/mês e 1 custom feature — suficiente para testar e validar
2. **Planos pagos** que diferem principalmente em:
   - Volume de agendamentos mensais
   - Número de custom features incluídas
   - Número de prestadores de serviço (staff)
   - Funcionalidades de compliance e enterprise
3. **Custom Features como add-ons** — o coração do modelo; o cliente escolhe quais features ativar dentro da cota do plano
4. **Créditos avulsos** para SMS, WhatsApp e IA de voz
5. **White Label** para agências e revendedores
6. **Serviços de onboarding** pagos (setup assistido)
7. **Marketplace Booking.page** — gratuito, sem comissão, como canal de aquisição

## Sistema de Custom Features

O sistema de Custom Features é o diferencial estratégico do SimplyBook.me. Em vez de um produto monolítico, a plataforma funciona como uma "loja" onde cada negócio monta sua stack de funcionalidades conforme sua necessidade e orçamento.

**Como funciona:**
- Cada plano define uma **cota** de custom features que podem ser ativadas
- O usuário acessa o painel de "Custom Features" e ativa as que deseja, dentro do limite do plano
- Algumas features são **gratuitas** (incluídas em qualquer plano sem consumir a cota)
- Features podem ser desativadas e trocadas por outras a qualquer momento
- A API expõe `isPluginActivated(featureName)` para checar o status de cada feature programaticamente

**Exemplo prático:** Um salão de beleza no plano Standard (8 custom features) pode ativar: Accept Payments + Instagram Bookings + Intake Forms + Coupons & Gift Cards + Client Login + Loyalty System + Service Add-ons + Membership. Um consultório médico trocaria Instagram e Loyalty por HIPAA + SOAP with Encryption.

### Agendamento Online

**Fluxo de self-scheduling do cliente:**

1. Cliente acessa a booking page (URL própria ou widget no site do negócio)
2. Seleciona a categoria de serviço (se configurado com Service Categories)
3. Seleciona o serviço desejado
4. Seleciona o prestador (ou "qualquer disponível" se habilitado)
5. Vê calendário com disponibilidade em tempo real (dias e horários)
6. Seleciona data e horário
7. Preenche formulário de intake (se configurado)
8. Preenche dados pessoais (nome, email, telefone)
9. Seleciona add-ons de serviço (se configurado)
10. Realiza pagamento online ou depósito (se configurado)
11. Recebe confirmação por email/SMS/WhatsApp
12. Recebe lembretes automáticos antes do horário

**Canais de booking disponíveis:**
- Página de booking própria (URL simplybook.me/en/company-name ou domínio customizado)
- Widget iframe embutido em qualquer site
- Botão de booking flutuante no site
- Contact widget (agendamento + mensagem)
- Smart Widget (React-based, para múltiplos serviços sequenciais)
- Facebook (botão "Book Now" na página do negócio)
- Instagram (link de booking no bio ou stories)
- Google Maps / Google Search (Reserve with Google)

## API e Webhooks

### Protocolo
- **JSON-RPC 2.0** sobre HTTPS
- Não REST nativo (embora haja menção a "REST" em alguns contextos — não confirmado como endpoint REST separado)

### Autenticação

| Tipo | Método | Headers |
|------|--------|---------|
| API Pública (cliente) | `getToken(companyLogin, apiKey)` | `X-Company-Login` + `X-Token` |
| API Admin | `getUserToken(companyLogin, userLogin, userPassword)` | `X-Company-Login` + `X-User-Token` |
| API de Aplicação | `getApplicationToken(applicationApiKey)` | — |

- **Token expira** em 1 hora — deve ser renovado
- Suporte a API User Keys (prefixo `api_user_key_`) para evitar uso de senha

### Endpoints principais

- **Login:** `https://user-api.simplybook.me/login`
- **API Pública:** `https://user-api.simplybook.me/`
- **API Admin:** `https://user-api.simplybook.me/admin`
- **Catálogo:** endpoint separado para listagens de marketplace

### Métodos disponíveis (seleção)

**Disponibilidade:**
- `getStartTimeMatrix(dateFrom, dateTo, serviceId, performerId, qty)` — slots disponíveis
- `getWorkCalendar(year, month, performerId)` — calendário de trabalho
- `getFirstWorkingDay()` — próxima data disponível
- `calculateEndTime(startDateTime, serviceId, performerId)` — calcula fim do agendamento
- `getAvailableUnits(eventId, dateTime, count)` — prestadores disponíveis

**Agendamentos:**
- `book()` — cria agendamento
- `confirmBooking(bookingId, signature)` — confirma (requer assinatura HMAC-MD5)
- `cancelBooking()` — cancela
- `rescheduleBook()` — reagenda
- `getBookings()` — lista agendamentos

**Dados:**
- `getEventList()` — lista serviços
- `getUnitList()` — lista prestadores
- `getAdditionalFields(eventId)` — campos do intake form
- `getCompanyInfo()` — dados da empresa
- `getClientList()` — lista clientes
- `addClient()` / `editClient()` — gestão de clientes

**Configuração:**
- `isPluginActivated(featureName)` — verifica se custom feature está ativa
- `getCompanyParam()` — parâmetros da empresa

### Segurança da API
- Assinatura segura: `md5(bookingId + bookingHash + secretKey)` para confirmação/cancelamento
- Secret API Key separada da API Key de autenticação

### Webhooks / Event-driven
- **Sem webhooks nativos** documentados
- **Via Zapier:** `pluginZapierSubscribe(url, notificationType)` suporta eventos:
  - `create` (novo agendamento)
  - `cancel` (cancelamento)
  - `change` (alteração)
  - `new_client` (novo cliente)
  - `create_invoice` (nova fatura)
  - `change_client` (dados do cliente alterados)
  - `delete_client` (cliente excluído)

### SDKs oficiais
- JavaScript (JSON-RPC client library)
- PHP (JsonRpcClient)

## Pontos Fracos / Gaps

1. **Interface administrativa complexa** — Quantidade de opções gera curva de aprendizagem acentuada; não é a mais moderna visualmente
2. **Custo por add-on acumula rápido** — Pequenas empresas que precisam de 5–8 custom features chegam rapidamente ao plano Standard/Premium
3. **Webhooks nativos ausentes** — Dependência do Zapier para event-driven integrations é limitação para devs avançados
4. **API apenas JSON-RPC** — Sem REST nativo; limita adoção por devs acostumados com REST; token expira em 1h (overhead de renovação)
5. **Relatórios limitados nos planos básicos** — Analytics avançados dependem de integrações externas (Google Analytics, etc.)
6. **Bugs e instabilidades reportadas** — Reclamações recorrentes de glitches, lentidão e comportamentos inesperados em booking
7. **Notificações não confiáveis** — 53% das menções negativas no Capterra citam falhas na entrega de emails e dificuldade de configuração
8. **Pagamentos limitados e confusos** — 42% das menções negativas citam configuração de pagamentos pouco clara
9. **Dados de clientes difíceis de localizar** — CRM não é a parte mais polida do produto
10. **Widget com personalização limitada** — Usuários relatam querer mais controle sobre aparência do widget externo
11. **Features podem conflitar** — Produto cresceu organicamente; algumas funcionalidades não se integram perfeitamente entre si
12. **Sem app próprio de staff** (além do admin geral) — Não há app separado para funcionários com visão reduzida
13. **Dependência de créditos para IA/SMS/WhatsApp** — Modelo de créditos avulsos pode surpreender usuários em volume alto
14. **Sem GraphQL** — API limitada a JSON-RPC; sem flexibilidade de queries customizadas

## O que Replicar para um Sistema Similar

### Modelo de produto
- **Sistema de add-ons/módulos ativáveis** é o modelo certo: permite pricing flexível, onboarding progressivo e evita feature bloat para clientes simples
- **Plano gratuito generoso** com limite de volume (não de tempo): converte melhor que trial de 14 dias
- **Marketplace próprio sem comissão** é aquisição de clientes barata e cria lock-in; replicar com `booking.page` equivalente

### Funcionalidades prioritárias (maior ROI)
- Accept Payments + Deposits — fundamental para reduzir no-show
- Intake Forms — coletar dados pré-consulta elimina trabalho manual
- Automated Reminders (SMS/WhatsApp) — maior impacto em no-show
- Client Login — essencial para fidelização e histórico
- Any Employee Selector — distribui carga automaticamente
- Calendar Sync (Google/Outlook) — necessidade básica de qualquer profissional
- Coupons & Gift Cards — ferramenta de marketing simples e eficaz
- Loyalty System — fidelização automatizada

### Modelo de canais
- Widget para sites externos (iframe) deve ser prioridade
- Integração com Google Business / Reserve with Google é canal de aquisição subestimado
- WhatsApp como canal de notificação supera SMS em taxas de abertura no Brasil

### Técnico
- API com webhooks nativos desde o início (evitar dependência do Zapier)
- Tokens de API sem expiração curta (1h é frustrante para integrações)
- Design system baseado em Figma com componentes reutilizáveis (como SimplyBook usa internamente)
- HIPAA/compliance como feature opcional — não obrigatória por padrão
- Multi-tenant com isolamento de dados por empresa desde a arquitetura
- Créditos para SMS/WhatsApp como receita adicional (modelo comprovado)

### Gaps a explorar (oportunidades de diferenciação)
- **CRM mais robusto** — SimplyBook.me tem CRM fraco; oportunidade de se diferenciar
- **Relatórios mais ricos** nativos (sem depender de Google Analytics)
- **App de staff separado** com visão limitada ao prestador individual
- **REST API nativa** além de JSON-RPC
- **Webhooks nativos** (sem Zapier)
- **Interface mais moderna e simples** — principal gap apontado em reviews
- **Integração nativa com WhatsApp Business API** (não apenas notificações)
- **Onboarding guiado** melhor — reclamação frequente de setup complexo
